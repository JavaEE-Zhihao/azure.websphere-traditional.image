IBM HTTP Server for WebSphere Application Server 9.0.5.x Image

Short description

This image includes the latest IBM HTTP server using IBM JDK 8 on RHEL 9.x

Description

This offer is an IBM HTTP Server for WebSphere Application Server 9.0.5.x Image and JDK 8 with Red Hat Enterprise Linux 9.x.

The VM instance that this offer creates is suitable for customers that require customized Azure deployments with a configurable IBM HTTP Server deployment. Users of this offer are expected to be familiar with IBM HTTP Server administration. The VM is installed with the specified GNU/Linux distribution, IBM JDK version, and IBM HTTP Server for WebSphere Application Server version. The relevant directories are as follows:

  • IHS_INSTALL_ROOT: /datadrive/IBM/WebSphere/IHS/V9
  • IHS_JAVA_HOME: ${IHS_INSTALL_ROOT}/java/8.0
  • PLUGINS_INSTALL_ROOT: /datadrive/IBM/WebSphere/Plugins/V9
  • PLUGINS_JAVA_HOME: ${PLUGINS_INSTALL_ROOT}/java/8.0
  • WCT_INSTALL_ROOT: /datadrive/IBM/WebSphere/Toolbox/V9
  • WCT_JAVA_HOME: ${WCT_INSTALL_ROOT}/java/8.0

IBM HTTP Server for WebSphere Application Server is not pre-configured. During VM creation, provide a username and SSH key or credentials for the admin account. After you log in with this account, do sudo -i to get write access to the relevant directories.

In addition to the VM IHS image, other offers are available to meet different needs. These offers include solution templates for nonconfigurable WebSphere Application Server Single Instance or Cluster in virtual machines and WebSphere Liberty/Open Liberty on AKS or ARO. Links to these offers are in the Learn more section at the bottom of the IBM WebSphere Product Family on Azure page.

If you want to work closely on your migration scenarios with the engineering team developing these offers, just click the CONTACT ME button on the IBM WebSphere Product Family on Azure page. Program managers, architects and engineers will reach back out to you shortly and initiate collaboration!
